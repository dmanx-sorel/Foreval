Исходники для GCC компилируются в CodeBlock16, CodeBlock17, CodeBlock20.
При компиляции в CodeBlock20 могут возникнуть препятствия.
Тестовая программа GCC была всё таки  скомпилирована в CodeBlock20, но не без трудностей.
Для запуска программы скопируйте Foreval.dll из папки .../Test/Delphi и поместите в папку с программой .../Test/GCC.
Для компиляции программы откройте файл  "ForevalG9.cbp" в  CodeBlock и скомпилируйте его.
Работает с Foreval.dll , скомпилированной как в Дельфи, так и в Лазарус.




                            Настройки программы: 

  модуль Types.h
    COMPLEX_VAR_ANY_ADDR - задаёт способ подключения комплексных переменных.
    EXTENDED_FLOAT - extended/double - задаёт тип переменных с плавающей точкой. Также и для функций.

  модуль ForevalG9Test.cpp
    переменная TypeCalcProc (= 1,2,3,4,5,6)  в процедуре test() задаёт способ вычисления введённых выражений.
    по умолчанию = 4; при запуске выбор = 2,4 (2 - самый быстрый, но без перехвата исключений)
                                              (4 - с полным перехватом всех исключений, но при этом, работает более медленно)


                     Последовательность выполнения тестов:

 1. Тестирование примеров возможностей Foreval. См исходники GCC.
 2. Сравнительные тесты производительности GCC vs Foreval для действительных чисел.
 3. Сравнительные тесты производительности GCC vs Foreval для комплексных чисел.
 4. Вычисление введённого выражения и первой производной по "x" от него.

PS
 После 3-го этапа будет предложен возврат обратно ко второму. 
 Для перехода к 4-му,  дважды ответьте "N".

                                     Комментарии к результатам сравнения:



1.
  Для комплексных чисел  вычисления выражений в Gcc идёт двумя способами
   GCC OVRL -  через задание типа комплексных чисел и перегрузку арифметических операторов для них. 
   GCC STD  -  через использование встроенного класса std::complex 

2.

 Сравнения производительности выполняются вычислением суммы выражения в цикле с изменением переменных, входящих в выражение.
 Приращение переменных задаётся как зависимость от введённого числа циклов,  чтобы оптимизатор не выкидывал выражение из цикла.
 Число циклов вычисления для каждого выражения устанавливается  в переделах 2E6-2E9  и для точности сравнения должно быть таким, 
  чтобы каждое выражение вычислялось не менее 10 сек.

PS
 Для GCC примеров в b.385+ был изменён способ вызова скомпилированных выражений. 
 Теперь (не для всех случаев) он выполняется через вызов прямой адресацией в коде (вставка inline)
 Работает на несколько % быстрей. Раньше - через вызовы вычисляющих Foreval  команд.
