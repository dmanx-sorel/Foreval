                                                      v.9.1.1.393

> Changed:              
              1. Most of the main functions with arrays have been rewritten and sped up several times: max, min, sum, sumq, prod,...

                 For the Integer, Single, Double array types, the calculation of these functions has been transferred to SSE.
                 
This imposes restrictions on the maximum intermediate value calculated in the function, corresponding to the array type.

                  (In previous versions, regardless of the array type, all calculations were in the FPU with the maximum value of the Extended type.)
                 
In connection with this, the minimum requirements for the processor have changed. SSE, SSE2, SSE3 support is required


                 Support for Integer arrays - only in the sum, max, min functions.

              2. Improved performance of the functions for calculating real polynomials specified by the  array - poly(Array, RealArg)

             

> Deleted:                  
              1. Support for the Integer array type in some of functions with arrays.

              2. The sumi function.
 

> Added:
              1. New auxiliary functions with arrays:
                  sumqs(VX,x)           =  sum ( (VX[i] - x)^2 ) 
                  prods(VX,x)           =  prod( (VX[i]-x) )
                  sumqs2v(VX,VY)        =  sum ( (VX[i]-VY[i])^2 )
                  sumprod2v(VX,VY)      =  sum ( (VX[i]*VY[i]) )
                  sumprods2v(VX,x,VY,y) =  sum ( (VX[i]-x)*(VY[i]-y) )

              2. Expanded argument types in array copy routines  CopyArray(VDst,VSrc): 
                   Single Array  -> Double Array; Extended Array
                   Integer Array -> Double Array; Extended Array
                   Double Array  -> Extended Array

> Fixed:      Numerical differentiation error in flCompileDerivATE,flCompileDiffExprATE commands.
       
> Changed:    Order of arguments in flCompileDerivATE command.

>  Some general code changes.
      
PS
        The current version of Foreval.dll is compiled in DelphiXE3.
        
When compiled in Delphi2009, the size will decrease by ~ 150kb.

                                                               v.9.1.1.375

> Added:
   
    1. Limited pointer arithmetic: 

        PointerVar                 = LineForm(PointerVars+IntegerVars)
        PointerVar                 = @Array[indx]
        PointerArray[indx]         = LineForm(PointerVars+IntegerVars)
        PointerArray[indx]         = @Array[indx]

    2. Reading and writing by pointer/address:
   
        PType(LineForm(PointerVars+IntegerVars))    
        PType(PointerArray[indx])  
   
       , where
          PointerVar - variable pointer. 
          PointerArray - array of pointers.
          LineForm(PointerVars+IntegerVars) - linear (extended) combination of pointers and integer variables.
          PType = type of read/write of pointer  = {PExtended, PDouble, PInteger, PSingle} 

    3. Copying arrays and memory sections of the corresponding types: 

          full copying of ArraySrc array to ArrayDst with length change ArrayDst (= ArraySrc ):
                ArrayDst = ArraySrc ; 
          
          full copy of ArraySrc to ArrayDst without changing the length of ArrayDst:
                CopyArray(ArrayDst , ArraySrc);  

          full copy of ArraySrc to ArrayDst (by pointer/address) without changing length ArrayDst:
                CopyArrayDbl, CopyArrayExt, CopyArrayInt, CopyArraySng (@ArrayDst, @ArraySrc) or (PointerArrayDst, PointerArraySrc) 
          
          partial copying (inserting) of an array section of the corresponding type with the starting address @AddrSrc at the address @AddrDst with  
             the number of cells - NumCopyCell:
                CopyMemDbl, CopyMemExt, CopyMemInt, CopyMemSng (@AddrDst,@AddrSrc,NumCopyCell)

    4.  High(Array) {Hi(Array)} - high(last) element of array  (=Length(Array)-1)

    5.  Commands for calling subroutines and returning from them from within the program: Gosub(LabelSub); Return();

    6.  Additional internal calculation commands with handling and interception of errors :
          SetMaskFPU();                  - masks exceptions FPU
          ErrorCode=ReadExceptionFPU();  - if there are exceptions, then returns '1', otherwise '0'
          ResetMaskFPU();                - resets exception flags
          ErrorCode=IsNAN(argf);         - returns '1' if argf is not a number (argf = NAN, ±INF) , otherwise '0'

        In the calculation cycle they work faster than try() - exception()     (See. doc)
   
    7. Additional external calculation commands with error interception:
          flSetExtAddrErrorFPU(@FPUError);  accepts the address of a variable to return an FPU error code when evaluating an expression in the following commands:
               flResultMaskedFPU(@CFunc);                                                     
               flResultMaskedFpuE(@CFunc, @Res);
               flResultMaskedFpuD(@CFunc, @Res);  
               flResultMaskedFpuCxE(@CFunc, @Re,@Im);                              
               flResultMaskedFpuCxD(@CFunc,  @Re, @Im);  
               flResultMaskedFpuCxEP(@CFunc,@Res);  
               flResultMaskedFpuCxDP(@CFunc,@Res);  

        In the calculation cycle they work faster than flResultSafe     (See. doc)                                

    8. flIsNAN(val:extended):integer - check val on NAN,INF  
   
    9.  flSetParamI(Name, IParam) - specifying integer parameters.
         
    10.  flCompileATE(Expr, Attr, var Addr, var RetType, var Error) - compiling an expression with return: address, result type, error

    11. Selecting a formula for calculating arccotan:
           standard:
              ŕrccotan(x) = pi/2 - arctan(x);  
           inverse:           
              arccotan(x) = arctan(1/x) ;  

> Changed !!!!!:
  
      
      1.      Swapped the location of the source and destination addresses in the memory copy auxiliary routines:  
                 flCopyArrayExt, flCopyArrayDbl, flCopyArrayInt, flCopyArrayExtDSC, flCopyMemory, flCopyVarCxD, flCopyVarCxE
                 flCopyVarD, flCopyVarE, flCopyVarI

               was: (AddrSrc, AddrDst,...);  now:  (AddrDst,  AddrSrc,...)

      2. If the added parameters, regardless of the type passed, are integers (Integer32),

>  Fixed: incorrect handling of calculation error - Float Point OverFlow in commands: Power, Exp,...

                                    v. 9.0.0.333

Foreval(v.9) is based on ForevalZ 
The  syntax core: of tree building, of optimizations,  and compilation was rewritten by ~ 70% to speed up parsing and reduce memory capture.
Incompatible with the previous version of Foreval(v.8) and ForevalZ, but supports common internal dll building logic.
Added Foreval(v.8)'s features that are not available in ForevalZ (not all):
  Parameters.
  Symbolic and numerical differentiation.
  Templates for symbolic differentiation of functions.
  Setting interpolation functions with one variable through cubic splines.
  More deeper optimization.

All supported operators and functions like Foreval(v.8)/ForevalZ.

Added (in relation to Foreval (v.8) / ForevalZ):
    Improved parsing and compilation of expressions (especially very long ones).
    Reduced memory capture on very long expressions.
    Added complex parameters.
    Added new optimization methods.   
    Minor optimization of symbolic differentiation.
 

Removed:
  Absent single float type.* 
  Removed ways to optimize Foreval (v.8) for analysis and replacement of typical expressions
     example: sh(a*(1-x)^2+1) vs sinh(1+sqr(-x+1)/0.5)      (a: parameter = 2)
     are considered the same in Foreval(v.8) and are optimized by replacing each other.
     In Foreval(v.9) these functions are recognized as different and are not optimized by replacing.
     Only functions with the same string representation are replaced.
  Some types of connection of external functions Foreval(v.8) are missing, presents only the main ones from ForevalZ.*
  
* (will be added in future versions)

PS
All subsequent versions will be compatible with v.9.0.0.xxx
